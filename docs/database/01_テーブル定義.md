# テーブル定義書（社員管理システム）

## 1. Users テーブル

### 1.1 概要

ログイン認証および権限管理に使用するユーザー情報を管理するテーブル。

### 1.2 カラム定義

| カラム名 | 型 | 必須 | PK | UNIQUE | 説明 |
|---|---|---|---|---|---|
| id | BIGSERIAL | ○ | ○ | - | ユーザーID |
| email | VARCHAR(254) | ○ | - | ○ | ログインメールアドレス |
| password | VARCHAR(60) | ○ | - | - | ハッシュ化済みパスワード |
| role | SMALLINT | ○ | - | - | ユーザー権限 |

### 1.3 制約

- PRIMARY KEY: `id`
- UNIQUE: `email`
- NOT NULL: `email`
- NOT NULL: `password`
- NOT NULL: `role`
- CHECK: `role IN (0,1)`

### 1.4 値制約

- `role`
  - 0：EMPLOYEE
  - 1：ADMIN
  
### 1.5 CRUD仕様

#### CREATE

- 新規ユーザー登録時にユーザー情報を登録する。
- 社員登録時に、対応するログインユーザー情報を登録する。

#### READ

- ログイン認証時にメールアドレスからユーザー情報を取得する。
- JWT認証時にユーザーIDからユーザー情報を取得する。

#### UPDATE

- 現時点ではユーザー情報の更新処理を実装しない。

#### DELETE

- 管理者による社員削除時に、対応するユーザー情報を物理削除する。


---

## 2. Employees テーブル

### 2.1 概要

管理者による社員情報の登録・更新・削除、CSVインポート、社員自身の情報取得・更新処理で使用する。

### 2.2 カラム定義

| カラム名 | 型 | 必須 | PK | UNIQUE | 説明 |
|---|---|---|---|---|---|
| id | BIGSERIAL | ○ | ○ | - | 社員ID |
| employee_number | VARCHAR(20) | ○ | - | ○ | 社員番号 |
| name | VARCHAR(50) | ○ | - | - | 氏名 |
| email | VARCHAR(254) | ○ | - | ○ | メールアドレス |
| hire_date | DATE | ○ | - | - | 入社日 |
| status | VARCHAR(20) | ○ | - | - | 在籍状態 |

### 2.3 制約

- PRIMARY KEY: `id`
- UNIQUE: `employee_number`
- UNIQUE: `email`
- NOT NULL: `employee_number`
- NOT NULL: `name`
- NOT NULL: `email`
- NOT NULL: `hire_date`
- NOT NULL: `status`

### 2.4 値制約

- `status`
  - `ACTIVE`（在籍）
  - `RETIRED`（退職）

### 2.5 CRUD仕様

- CREATE
  - 管理者による社員情報の新規登録
  - CSVインポートによる社員情報の一括登録

- READ
  - `ADMIN`：社員情報を全件取得する。
  - `EMPLOYEE`：自身の社員情報を取得する。
  - `EMPLOYEE`：社員一覧では氏名・メールアドレスのみ取得する。

- UPDATE
  - `ADMIN`：社員情報を更新できる。
  - `EMPLOYEE`：自身の氏名・メールアドレスのみ更新できる。
  - CSVインポート時に既存社員情報を更新できる。

- DELETE
  - 管理者のみ社員情報を物理削除できる。

### 2.6 更新ルール
  - id(PK)を検索キーとし、該当するリストの情報を更新する
  - id(PK)を関連キーとし、Userテーブル、Employeeテーブルに登録を行う
  - CSVインポート時も id(PK) を基準に登録・更新を判定する

---

## 3. Department テーブル

### 3.1 概要

部署を管理するテーブル。

### 3.2 カラム定義

| カラム名 | 型 | 必須 | PK | UNIQUE | 説明 |
|---|---|---|---|---|---|
| id | BIGSERIAL | ○ | ○ | - | 部署ID |
| department | VARCHAR(50) | ○ | - | - | 部署 |

### 3.3 制約

- 新規登録時
- NOT NULL: `department`

### 3.4 値制約

- `department`
  - `GENERAL`（総務）
  - `HR`（人事）
  - `PLANNING`（企画）
  - `SALES`（営業）

### 3.5 CRUD仕様

- CREATE
  - 管理者による部署情報の新規登録
  - CSVインポートによる社員情報の一括登録

- READ
  - `ADMIN`：部署情報を取得する。
  - `EMPLOYEE`：部署情報を取得する。
  - `EMPLOYEE`：社員一覧の場合も部署情報を取得する。

- UPDATE
  - `ADMIN`：部署情報を更新できる。
  - CSVインポート時に既存社員情報を更新できる。


### 3.6 更新ルール
  - id(PK)を検索キーとし、該当するリストの情報を更新する
  - id(PK)を関連キーとし、Departmentテーブルに登録を行う
  - CSVインポート時も id(PK) を基準に登録・更新を判定する

## 5. テーブル間の関連

Usersテーブル、Employeesテーブル、Departmentテーブル、
Positionテーブルは、それぞれのidをキーとしてアプリケーション側で関連付ける。
