# テーブル定義書（社員管理システム）

## 1. users テーブル

### 1.1 概要

ログイン認証および権限管理に使用するユーザー情報を管理するテーブル。

### 1.2 カラム定義

| カラム名 | 型 | 必須 | PK | UNIQUE | 説明 |
|---|---|---|---|---|---|
| id | BIGSERIAL | ○ | ○ | - | ユーザーID |
| email | VARCHAR(254) | ○ | - | ○ | ログインメールアドレス |
| password | VARCHAR(60) | ○ | - | - | ハッシュ化済みパスワード |
| role | VARCHAR(20) | ○ | - | - | ユーザー権限 |

### 1.3 制約

- PRIMARY KEY: `id`
- UNIQUE: `email`
- NOT NULL: `email`
- NOT NULL: `password`
- NOT NULL: `role`

### 1.4 値制約

- `role`
  - `ADMIN`
  - `EMPLOYEE`

### 1.5 CRUD仕様

- CREATE  
  新規ユーザー登録時にユーザー情報を登録する。

- READ  
  ログイン認証時およびJWT認証時にユーザー情報を取得する。

- UPDATE  
  現時点では更新処理を実装しない。

- DELETE  
  管理者による社員削除時に、対応するユーザー情報を物理削除する。

## 2. employees テーブル

### 2.1 概要

社員情報を管理するテーブル。

管理者による社員一覧画面からの登録・更新・削除、CSVインポート、社員自身の情報取得・更新処理で利用する。

### 2.2 カラム定義

| カラム名 | 型 | 必須 | PK | UNIQUE | 説明 |
|---|---|---|---|---|---|
| id | BIGSERIAL | ○ | ○ | - | 社員ID |
| employee_number | VARCHAR(20) | ○ | - | ○ | 社員番号 |
| name | VARCHAR(50) | ○ | - | - | 氏名 |
| email | VARCHAR(254) | ○ | - | ○ | メールアドレス |
| department | VARCHAR(50) | ○ | - | - | 部署 |
| position | VARCHAR(50) | ○ | - | - | 役職 |
| hire_date | DATE | ○ | - | - | 入社日 |
| status | VARCHAR(20) | ○ | - | - | 在籍状態 |

### 2.3 制約

- PRIMARY KEY: `id`
- UNIQUE: `employee_number`
- UNIQUE: `email`
- NOT NULL: `employee_number`
- NOT NULL: `name`
- NOT NULL: `email`
- NOT NULL: `department`
- NOT NULL: `position`
- NOT NULL: `hire_date`
- NOT NULL: `status`

### 2.4 値制約

- `department`
  - `GENERAL`（総務）
  - `HR`（人事）
  - `PLANNING`（企画）
  - `SALES`（営業）

- `position`
  - `STAFF`（一般社員）
  - `LEADER`（主任）
  - `MANAGER`（課長）
  - `DIRECTOR`（部長）
  - `EXECUTIVE`（本部長）
  - `CEO`（社長）

- `status`
  - `ACTIVE`（在籍）
  - `RETIRED`（退職）

### 2.5 CRUD仕様

- CREATE
  - 管理者による社員情報の新規登録
  - CSVインポートによる社員情報の一括登録

- READ
  - `ADMIN`：社員情報を全件取得する。
  - `EMPLOYEE`：自身の社員情報を取得する。
  - `EMPLOYEE`：社員一覧では氏名・メールアドレスのみ取得する。

- UPDATE
  - `ADMIN`：社員情報を更新できる。
  - `EMPLOYEE`：自身の氏名・メールアドレスのみ更新できる。
  - CSVインポート時に既存社員情報を更新できる。

- DELETE
  - 管理者のみ社員情報を物理削除できる。

### 2.6 更新ルール

- `employee_number` が既に存在する場合は、既存の社員情報を更新する。
- `employee_number` が存在しない場合は、新規社員として登録する。
- CSVインポート時も、`employee_number` を基準に登録または更新を判定する。
- 管理者が社員一覧から対象社員を選択した場合、選択された社員の `employee_number` を用いて更新対象または削除対象を特定する。
